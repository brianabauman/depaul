Non Goals

  • Learn an IDE

Machines and Programs

  • Machine: responds to inputs and generates outputs
    • Batch computing: input at beginning, output at end
    • Interactive computing: input/output interleaved
  • Program: tells machine how to respond to input and outputs
    • Earliest machines programmed by moving wires
    • Stored program computer (aka Von Neumann Machine)

Computing = control + memory

  • Programs can be seen as taking inputs and generating outputs
  • How to specify the mapping from to output?
    • control: the process used to generate output
      typically fixed before computation begins
    • memory: space for input, output, intermediate results
      typically varies during computation
  • Control
    • imperative procedure, mathematical function, logical specification
  • Memory
    • global variables, local variables, others

Time

  • Programs can be transformed at different times
    • Compile time = before execution = static = early
    • Run time = during execution = dynamic = late
  • Finer grained
    • Compiler: Compile, link, load
    • Memory: Allocation, initialization, deallocation
    • Function: call, return
  • Even finer
    • Machine instruction: issuance, pipeline stages, commit, ...
    • Machine memory: cache flush, virtual memory paging, ...
    • System: interrupts, process swaps, ...

Memory

  • Major challenge for programmers is memory management
  • Major task for PL is to support memory management
  • Copying versus sharing
  • Allocation/deallocation: many categories
    • Global variables, live forever
    • Local variables, for functions and methods
    • Unmanaged (manually allocated/deallocated)
    • ARC (automatic reference counting)
    • Garbage collected

Interpretation and Translation

  • Machine has a language for programs
    • x64 binary, Javascript, Java Virtual Machine Language (JVML), Common Intermediate Language (CIL) etc...
  • Machine is an interpreter for programs
    • x64 processor, Javascript engine, Java virtual machine (JVM), Common Language Runtime (CLR), Low-Level Virtual Machine (LLVM)
    • Machine = interpreter = runtime system = engine
  • Program may be transformed into an equivalent program
    • Javascript to javascript, Java to JVML, JVML to LLVM, LLVM to x64 binary
    • Translator = Compiler = Transpiler

C# execution on i7

  • Compiler translates C# to CIL
  • CLR loads CIL for initial class
  • CLR translates CIL for main function to x64 binary
  • x64 process translates x64 binary to microcode
    • Microcode actually executes!

Designing a language

  • Resource (time/space/etc) constraints: compilation? execution?
  • Programmer model: functionality and performance (time/space)
    • Programmer complexity versus predictability
    • Automatic conversions and coercions: Autoboxing? String to number?
  • Error/debugging model
    • When do errors occur? Static versus Dynamic
    • What is an error?
    • Errors/Undefined behavior/undefined values/defined values
  • Modularity/Reusability
  • Versioning: eg, Java String

Syntax versus semantics

  • Syntax: Can I run it?
  • Semantics: What will it do?
  • Pragmatics: How many resources (time/space/etc) will it consume?
